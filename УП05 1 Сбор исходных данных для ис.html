УП05 1 Сбор исходных данных для ис

1.Основные группы исходных данных

Организационно-экономические данные включают:

  • Уставные документы: название организации, форма собственности, структура управления
  • Бизнес-процессы: текущие процессы, роли участников, документооборот, регламенты
  • Проблемы и цели: существующие недостатки, целевые показатели
  • Экономические параметры: бюджет, сроки окупаемости

Технические данные охватывают:

  • Инфраструктуру: аппаратное и программное обеспечение
  • Инфраструктуру: аппаратное и программное обеспечение
  • Технические характеристики: производительность, надежность, безопасность

Функциональные требования содержат:

  • Сценарии использования: описание действий пользователей
  • Функционал системы: справочники, документы, отчеты
  • Бизнес-логику: правила обработки данных

Пользовательские требования определяют:

  • Целевую аудиторию: уровень подготовки
  • Эргономику: удобство использования
  • Дизайн: стилистические решения

Нормативно-правовая база включает:

  • Законодательные требования: хранение документов, ЭДО
  • Отраслевые стандарты: ГОСТы, регламенты

2.Методы сбора данных

  • Интервью с ключевыми лицами
  • Анкетирование для массового сбора данных
  • Наблюдение за рабочими процессами
  • Анализ документации
  • Workshops для коллективной работы
  • Прототипирование для визуализации
  • Brainstorming для генерации идей

3.Процесс сбора данных

Подготовительный этап:

  • Определение целей
  • Составление списка стейкхолдеров
  • Разработка плана-графика
  • Подготовка инструментов сбора

Этап сбора:

  • Проведение интервью
  • Сбор документации
  • Фиксация технических параметров
  • Документирование всех данных

Анализ и систематизация:

  • Обработка собранной информации
  • Выявление противоречий
  • Построение схем процессов
  • Структурирование требований

Формирование отчета:

  • Подготовка документации
  • Согласование с заказчиком
  • Внесение корректировок

4.Типичные ошибки

  • Игнорирование мнений рядовых пользователей
  • Противоречивые требования без анализа
  • Отсутствие формального согласования
  • Односторонний подход к сбору данных

5.Рекомендации

  • Учитывать мнение всех заинтересованных сторон
  • Использовать комплексный подход к сбору данных
  • Проводить тщательный анализ противоречий
  • Документировать все этапы работы
  • Обеспечивать формальное согласование результатов

Вывод: Качественно собранные исходные данные являются фундаментом успешного проекта по созданию информационной системы. Их сбор требует системного подхода, профессионализма и внимательности к деталям.
